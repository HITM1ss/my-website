## 给播放器补一个小修复

测试时还发现了一个小问题：首页加载后，歌单的第一首歌偶尔无法播放；手动切到下一首却又一切正常。

原因是播放器会先通过 **Meting API** 获取歌单，再由第三方接口解析每首歌真正的播放地址。第一首歌的解析请求偶尔没有及时响应，于是就卡在了原地。

于是我给播放器加了一个小小的“容错机制”：当一首歌加载失败时，它会自动尝试播放下一首。这样即使遇到偶发失效的链接，也不用手动切歌啦。

### 它是怎么工作的？

1. 每次加载歌曲时，使用 `loadVersion` 标记当前这一次请求。
2. 监听浏览器音频元素的 `error` 事件。
3. 如果歌曲加载失败，就记录它在歌单中的位置，并寻找下一首尚未失败的歌曲继续播放。
4. 如果整张歌单都尝试过仍然无法播放，就停止切换，避免无限循环。
5. 手动切歌、上一首或重新选择歌曲时，会清空本轮失败记录，让这些歌曲之后还有机会重新播放。
6. `lastHandledErrorVersion` 则确保同一首歌重复触发错误时，只会处理一次。

核心逻辑很短，大致是这样：

```ts
audio.addEventListener("error", function () {
	if (lastHandledErrorVersion === loadVersion) return;
	lastHandledErrorVersion = loadVersion;

	state.failedTrackIndexes[state.currentIndex] = true;
	var nextIndex = state.currentIndex;

	for (var i = 0; i < state.playlist.length; i++) {
		nextIndex = (nextIndex + 1) % state.playlist.length;
		if (!state.failedTrackIndexes[nextIndex]) {
			loadTrack(nextIndex, true, true);
			return;
		}
	}
});
```
